# 快速开始指南

本指南将帮助你快速搭建并运行 Bun CodeView 系统。

## 1. 环境配置

### 安装 Bun
Bun 是运行本系统所必需的运行时及包管理器。
```bash
curl -fsSL https://bun.sh/install | bash
```

### 克隆项目
```bash
git clone <repository-url>
cd bun-codeview
```

## 2. 安装与运行

### 一键安装所有依赖
在根目录下执行：
```bash
bun install
```

### 启动开发环境

执行以下命令即可启动 Tauri 桌面应用：

```bash
bun tauri dev
```
> 这将同时启动 Rust 后端和 Vue 前端，并打开桌面窗口。

## 3. 基本使用流程

1.  **启动应用**：运行 `bun tauri dev` 后，桌面应用窗口将自动打开。
2.  **配置路径**：在侧边栏的项目路径输入框中输入你想要分析的本地项目绝对路径，或点击图标浏览目录。
3.  **开始分析**：点击“开始分析”按钮，系统将执行原生 AST 扫描并构建依赖图谱。
4.  **文件检索**：在左侧文件树顶部输入关键词，可实时搜索并高亮定位项目文件。
5.  **查看与编辑**：
    - 点击文件即可打开。对于代码文件，支持 100+ 语言的高亮展示。
    - 对于 `.md` 文件，可在右上角切换“预览”与“源码”。
    - 在源码模式下，支持表格自动补全（Enter 键）及图片粘贴/拖拽（自动存入 `attachments`）。
6.  **保存更改**：编辑后点击右上角“保存”或按下 `Cmd/Ctrl + S` 将修改直接存回磁盘。
7.  **查看图谱**：你可以缩放、拖拽生成的架构图。双击节点可向下钻取层级结构（分层视图）。

## 4. 常见问题 (FAQ)

- **问：支持分析哪些语言？**
  - 目前原生支持 TypeScript (`.ts`, `.tsx`), JavaScript (`.js`, `.jsx`) 以及 Java (`.java`)。
- **问：分析大型项目会卡顿吗？**
  - 本系统采用 Tree-sitter 高效解析，并在内存中完成图计算。对于数十万行级别的代码，分析通常在几秒内完成。
- **问：如何更新规则？**
  - 你可以在 `backend/src/governance/RuleEngine.ts` 中添加自定义的架构治理规则。
