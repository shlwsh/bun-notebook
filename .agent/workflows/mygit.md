这个工作流用于将当前会话的代码变更汇总并提交到 Git 远程仓库。

## 📋 工作流程

1. **回顾变更内容**：仔细回顾本次 AI 会话中对业务代码、功能实现的修改内容。
2. **AI 汇总小结**：将这些变更汇总为一段简洁且专业的**中文**提交信息（总结业务价值，而非仅仅列出文件名）。
3. **执行提交**：运行以下命令完成 Git 提交和推送。

## 🚀 执行命令

```bash
bun run mygit "AI-Summary: [在这里填入您的中文业务变更汇总]"
```

## 💡 提交信息格式建议

提交信息应包含以下要素：
- **业务价值**：说明这次变更解决了什么问题或带来了什么价值
- **变更范围**：简要说明涉及的功能模块或文件
- **变更类型**：如「新增功能」、「优化改进」、「修复问题」等

### 示例

```
bun run mygit "新增发布部署文档章节，完善项目说明"
```

```
bun run mygit "优化 README 文档结构，添加详细的发布部署和分发指南"
```

```
bun run mygit "完善知识库服务功能，支持文档导入和分块存储"
```

## ⚙️ 脚本功能说明

`mygit.ts` 脚本会自动执行以下操作：
1. 检查 Git 状态，确认有待提交的变更
2. 使用提供的提交信息（或自动生成兜底信息）
3. 执行 `git add .` 添加所有变更
4. 执行 `git commit` 提交变更
5. 执行 `git push` 推送到远程仓库

## 📝 注意事项

- 执行前请确保已完成所有代码修改并在终端验证通过
- 提交信息应使用中文，简洁明了地描述变更的业务价值
- 如果未提供提交信息参数，脚本会自动根据文件名生成兜底信息
