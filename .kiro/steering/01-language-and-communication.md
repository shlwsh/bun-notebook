# 语言和交互规则

## 交互语言要求

**所有与用户的交互必须使用中文**，包括但不限于：

- 需求讨论和确认
- 技术方案说明
- 代码注释和文档
- 错误信息和提示
- 进度汇报和总结

## 文档语言要求

**所有生成的文档必须使用中文**，包括：

- 需求文档
- 设计文档
- 技术方案
- API 文档
- 用户手册
- 开发日志
- 测试报告

## 代码注释规范

- 关键函数和复杂逻辑必须添加中文注释
- 公共 API 和接口必须有中文说明
- 类型定义和接口需要中文描述
- 配置文件中的重要配置项需要中文注释

## 命名规范

- 代码中的变量、函数、类名等使用英文（遵循各语言规范）
- 文档文件名可以使用中文
- Git commit 消息使用中文
